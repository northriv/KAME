Layer 1: STM Commit

TLA+ Model Explained — stm_commit.tla (156 lines)

Premise: The atomic_shared_ptr verified in Layer 0 is abstracted as a correct atomic register. Details of reference counting are not modeled.

Modeled Cycle

Snapshot → Write → Commit(CAS) → Success or Retry

Variables (7) — Very Simple

VARIABLES
    node_val,       \* Payload value of the shared node
    node_serial,    \* Serial number (incremented by 1 per commit)
    pc,             \* "idle", "snapshot", "write", "commit", "done"
    thr_snap_val,   \* Value at snapshot time
    thr_snap_ser,   \* Serial at snapshot time
    thr_write_val,  \* Value to be written
    thr_committed   \* Number of successful commits

Significantly simpler than the 12 variables in Layer 0. This is the power of abstraction.

iterate_commit Cycle

Step 1: Snapshot

TakeSnapshot(t) ==
    /\ pc[t] \in {"idle", "done"}
    /\ thr_snap_val' = [thr_snap_val EXCEPT ![t] = node_val]
    /\ thr_snap_ser' = [thr_snap_ser EXCEPT ![t] = node_serial]
    \* → 1 atomic load (verified in Layer 0)

Step 2: Write (2 variants modeled)

(* 2a: Non-deterministic write — arbitrary closure *)
Write(t) ==
    /\ \E v \in 0..MaxVal :
       thr_write_val' = [thr_write_val EXCEPT ![t] = v]

(* 2b: Deterministic increment — tr[node].x += 1 *)
WriteIncrement(t) ==
    /\ thr_write_val' = [thr_write_val EXCEPT
         ![t] = thr_snap_val[t] + 1]

Step 3: Commit (CAS)

Commit(t) ==
    \/ (* CAS success: both value and serial match the snapshot *)
       /\ node_val = thr_snap_val[t]
       /\ node_serial = thr_snap_ser[t]
       /\ node_val' = thr_write_val[t]       \* Install new value
       /\ node_serial' = node_serial + 1     \* Serial +1
       /\ thr_committed' = [thr_committed EXCEPT ![t] = @ + 1]

    \/ (* CAS failure: another thread already committed → retry *)
       /\ ~(node_val = thr_snap_val[t] /\ node_serial = thr_snap_ser[t])
       /\ thr_snap_val' = [thr_snap_val EXCEPT ![t] = node_val]
       /\ thr_snap_ser' = [thr_snap_ser EXCEPT ![t] = node_serial]
       /\ pc' = [pc EXCEPT ![t] = "snapshot"]

Key point: On CAS failure, a new snapshot is taken immediately and the operation is retried. This is the behavior of iterate_commit.

Verified Invariants (5 + TypeOK)

\* 1. No Lost Update: if 2 threads both commit successfully → serial increases by at least 2
NoLostUpdate ==
    \A t1, t2 \in Threads :
        (t1 /= t2 /\ thr_committed[t1] > 0 /\ thr_committed[t2] > 0)
        => node_serial >= 2

\* 2. Commits are serialized: total commit count across all threads ≤ serial
CommitSerializes ==
    totalCommits <= node_serial

\* 3. Snapshot precedes commit: committer's snap_ser < current serial
SnapshotBeforeCommit ==
    \A t \in Threads :
        (pc[t] = "done" /\ thr_committed[t] > 0)
        => thr_snap_ser[t] < node_serial

\* 4. Value is within range
ValueBounded == node_val <= MaxVal

\* 5. The last committer's written value is visible
WriteReadConsistency ==
    \A t \in Threads :
        (pc[t] = "done" /\ thr_snap_ser[t] = node_serial - 1)
        => node_val = thr_write_val[t]
